Czym w ogóle jest JAVA?

Najprościej mówiąc JAVA jest jednym z najbardziej popularnych język programowania zorientowanym obiektowo.

Java jest językiem ogólnego zastosowania co oznacza że można ją spotkać prawie wszędzie.

Przykładami mogą tu być:

  • serwery aplikacyjne
  • aplikacje mobline
  • systemy bankowe
  • aplikacje korporacyjne

To jedynie kilka z przykładowych gałęzi zastosowań języka, w rzeczywistości jest tego znacznie więcej.

Krótko o historii

Twórcami języka są James Gosling, Mike Sheridan i Patric Naughton z firmy Sun Microsystems. Początkowo język powstawał w ramach projektu Green po nazwą OAK.

W 1995 roku w San Francisco na konferencji SunWorld język został publicznie zaprezentowany pod nazwą Java. W tym samym roku firma Sun wydała pierwszą publiczną implementację Java 1.0.

Nastęnie w roku 1996 pod kątem webowym pojawiła się przeglądarka Netscape 2.0, kótra to była kompatybilna z Javą 1.0. Co niewątpliwie przyczyniło się do wzrostu zainteresowania językiem.

Kolejne wydanie języka tym razem w 1998 roku firma publikuje Java 2 SE (Standard Edition) z około 2300 klasami. Następnie w tym samym roku wypuszczona zostaje Java 2 EE (Enterprise Edition) oraz Java 2 ME (Micro Edition). J2EE zawierała technologie oraz interfejsy API przeznaczone dla środowisk koroporacyjnych czy serwerowych. W odróżnieniu od J2EE J2ME zawierała interfejsy zoptymalizowane pod kątem aplikacji mobilnych.

W 2006 roku powstaje Java FX przeznaczona głównie do tworzenia aplikacji desktopowych.

W 2010 roku firma Oracle przejmuje Javę. Następne lata to kolejne wersje języka. Nie będę wymieniać tu wszystkich ale, o których warto wspomnieć to na pewno te z LTS (long-term support - długi okres wsparcia), czyli Java SE8 z 2014 roku, Java SE11 oraz przyszła wersja, która zostanie wydana we wrześniu bieżącego roku (2021).

Dobrze, po tym krótkim i ogólnym wstępie pora przejść do zagadnień związanych już z samym programowaniem oraz z tym jak to się odbywa. Tak więc zaczynajmy.

Kluczowe koncepcje języka

Obiektowość

Obiektowość oznacza że programy napisane za pomocą Javy są zbiorami współpracujących obiektów. Gdzie każdy obiekt posiada swój stan i metody, które wykonuje oraz wymienia informacje z innymi obiektami w celu osiągnięcia założonego przez programistę efektu.

Obiekt natomiast, w rozumieniu twórców to każdy dający się wyodrębnić element rzeczywistości tak samo fizyczny jak i abstrakcyjny. Dla przykładu obiektem może być samochód, zwierze, człowiek, konto użytkownika w serwisie czy emocje.

Dziedziczenie

Dziedziczenie jest mechanizmem, który pozwala obiektom na dosłowne odziedziczenie zachowań i stanów obietków położonych wyżej w hierarchii. Można powiedzieć że obiekt-dziecko dziedziczy/przejmuje cechy po obiekcie-rodzic. To bardzo przydatny mechanizm gdyż czasami chcąc tworzyć obiekty które różnią się od siebie jedynie jednym stanem czy metodą a poza tym są takie same ten mechanizm oszczędzanam sporo pracy, zaoszczędza pamieć komputera i przyśpiesza wykonywanie się programu.

Niezależność od architektury

Właściwość ta jest osiągana przez Javę dzięki temu że kod źródłowy jest kompilowany do kodu pośredniego. Kod pośredni inaczej kod bajtowy (ang. bytecode) ma tę właściwość że jest niezależny od systemu operacyjnego czy procesora. Jedyne czego system operacyjny potrzebuje aby wykonać program Javy jest Wirtualna Maszyna Javy - JVM (ang. Java Virtual Maschine). JVM w uproszczeniu dostosowuje kod bajtowy do specifiki systemu operacyjnego oraz procesora.

Niezawodność i bezpieczeństwo

Jeśli chodzi o te kwestie autorzy języka zrobili naprawdę dużo. Co mam na myśli? Pierwszym punktem jest mechanizm wyjatków. System ten jest uruchamiany w momencie gdy aplikacja natrafi na nieprzewidziane trudności takie jak np.

  • operacja na pustym elemencie
  • operacje poza zadeklarowaną granicą tablicy
  • podanie nieprawidłowych danych użytkownika
  • dzielenie przez zero
  • odczytywanie niedostępnego pliku czy adresu URL

oraz w wielu innych przypadkach.

Innym przykładem niezawodności jest system logowania, który pozwala na zapisywanie w plikach dziennika przebieg działania programu.

Natomiast z perspektywy bezpieczeństwa istnieje system asercji, który pozwala upewnić się że wymagania co do określonych wyrażeń są prawdziwe.

Sieciowość

Sieciowość w Javie jest traktowana jako odpowiednio przygotowane klasy. Przez trzymanie się zasad obiktowości Java nie widzi różnicy czy to jest plik lokalny czy dane z sieci. To też ułatwia pracę programistom.
